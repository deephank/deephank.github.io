聊天機器人不只能一問一答，使用有限狀態機的思考方式，我們可以很容易的設計出情境式的聊天機器人。

聊天機器人

傳統的聊天機器人是分析 Q 之後，藉由撈資料庫、爬蟲、自動生成等等，回答出 A

但人類對話時，並不是一問一答的形式。我們會延續前一句話、圍饒在一個話題。當我接著問「明天呢？」，我們期待對方會延續前句話的情境繼續說下去。

有限狀態機

有限狀態機，Finite State Machine(FSM)，是表示有限個狀態以及在這些狀態之間的轉移和動作等行為的數學模型。

如：門的行為可以畫成有限狀態機，如果門開著(狀態)，可以由關起來(動作)，讓門的狀態變成「關著」的。

情境聊天機器人設計

藉由「有限」、「狀態」等特性，我們可以模仿 FSM 設計出一個情境的聊天機器人。我們把剛剛的對話畫成一張有向強連通圖。

我們將每次對話的 state 存起來(預設為default)，在下一個問題進來時，從該 state 出發，判斷跳往下一個 state，然後透過各別的方式生成答案。

討論

有限的狀態

如果使用這個架構，我們必須把行為設定在有限的情境裡面，所以如果要做出接近人類的機器人，我們需要先設計非常多的「情境」。

另一個方面，由於我們已經在某一個情境中，所以我們可以縮小範圍，專注於設計這個情境的演算法：如「天氣：爬蟲」、「問答：用ML方法生成」、「知識：撈資料庫」⋯⋯

判斷器

畫出來的圖是一張強連通圖，任何一個Vertice都有 in&out。且任何 Question 都必須能跳往某個情境(if-elif-else)。

製作良好的判斷器是非常難的，我們希望「今天會下雨嗎」、「今天天氣如何」、「今天熱嗎」代表的是同一件事，本篇的範例是用 if Q==something的方式，顯然這非常不好用。

所以必須額外用各種自然語言處理的方法兜出一個好的判斷器，不論是斷詞、TFIDF、餘弦相似、word embedding、RNN…都是需要花很多時間實驗的。
